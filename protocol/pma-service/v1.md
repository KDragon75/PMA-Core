# PMA service protocol v1

PMA protocol version 1 uses one JSON-RPC 2.0 object per UTF-8 line over stdio. Clients negotiate with `pma.initialize` using an inclusive integer range:

```json
{"protocol_version":{"min":1,"max":1},"client":{"name":"example","version":"1.0"}}
```

Task 003 methods are `pma.initialize`, `pma.health`, `pma.get_capabilities`, `pma.get_status`, `pma.shutdown`, and the `$/cancelRequest` notification. Unknown fields are tolerated. Request IDs are strings or integers and may not be reused during one process lifetime.

JSON-RPC numeric errors retain standard meanings. Stable PMA error fields are under `error.data`; internal exception and storage messages are never included.
